1. The Box Model
	-  everything can be viewed as a box with the following contents - the main content of the element (content box); the inner padding of the element (padding box); the borders of the element (border box) and the margins of the element, which grow outwards and push away other elements (margin box)
	- margin collapse - the greater margin of two neighboring elements is applied
	- block and inline elements:
		- block - block elements go from top to bottom, breaking onto a new line and respecting all margins; they cant be put next to each other conventionally as they strive to take all available width on their line - i.e. they push away other elements; width and height can be set
		- inline - they do not break into a new line; they cannot accept width or height and while respecting paddings and inline margins (left-right) they do not push other elements away
2. Selectors 
	- id selector - applied to all elements with the respective id attribute
	- type selector - apply to all elements of the given type
	- class selector - apply to elements with the respective class attribute
	- general selectors - apply to all elements
3. Specificity and cascading
	- inheritance - there are several inheritable properties such as color, which, if not explicitly defined, will be sourced from the first parent that has them set
	- cascading - defines which styles will be imported when considering user-agent styles, user and author (developer) stylesheets, etc.
	- specificity - styles can be combined across different stylesheets, in which case special algorithmic rules apply, which decide how to reconcile conflicting styles (most common of them is specificity);
		- selectors have a given specificity - id > class > type
		- when combining selectors those with higher specificity take precedence - ex. id + class has higher specificity than two classes
		- in case of equal specificity the latest selector is applied
		- inline styles - have the highest precedence, which is why they are an anti-pattern
		- !important - overrides specificity rules, but breaks the algorithm; to be avoided and used as a last resort only (99% off the cases increasing selector specificity will work out for you)
		- making your selectors more specific via combinators -  .class1 > p for example
			- descendant selector - ul li a - an anchor within a list item in an unordered list
			- direct descendant selector - div.text > p - matches only direct children elements of the element specified; it won't affect nested tags
			- adjacent sibling combinator - img + p - matches the second element only if it directly follows the first and both are descendants of the same parent
			- general sibling combinator - p ~ code - will match all iterations of the second element if they come after the fist and have the same parent (they do not have to directly follow them)
			- **combinators do not affect specificity; only the selectors used in them do**
	```html
	<body>
	  <header>I'm a header</header>
	  <main  class="main">
	    <p class="myText">This is the main content of the website</p>
	  </main>
	</body>

```
	
	``` scss
	p { // margin and font-weight properties will be inherited
	  margin: 0;
	  font-weight: bold;
	  color: red;
	}

	.main > p { // direct descendent, specific
	  color: purple;
	}

	main p { // any descendent, not as specific
	  color: blue;
	}

	.myText { // won't be applied due to first combinator 
	  color: green;
	}
```

4. Positioning 
	- document flow - defined by how we lay out and order our elements
	- z-index and the stacking context - the stacking context determines how elements are layered on top of each other alongside an imaginary z-axis; this starts to become more relevant as we build elements such as modals, popups, alerts and is directly related to how elements behave to each other once we start adjusting their position property; multiple stacking contexts can exist simultaneously and independently of each other
	- static positioning - the normal position of an element
	- absolute and relative positioning
		- relative elements will still appear as normal, although they will accept changes to their left, top, bottom, right properties; still inside of the document flow and can accept a z-index value
		- absolutely positioned elements will position themselves to the nearest relative parent and according to the provided offset; removed from the document flow and creates a stacking context if z-index is set; if no z-index is specified, the last absolutely positioned child will have a greater z-index value
	```html
	<section>
		<p>Demonstration of relative, absolute and fixed positioning</p>
		 <div class="parent">
		    <div class="child"></div>
		    <div class="child2"></div>
		    <div class="child3"></div>
		</div>
	</section>
```

	```scss
	.parent {
	  position: relative;
	  width: 250px;
	  height: 250px;
	  //height: 100vh;
	  background-color: blue;
	}

	.child {
	  position: absolute;
	  // position: fixed;
	  width: 50px;
	  height: 50px;
	  background-color: red;
	  // inset: 0;
	  top: 125px;
	  left: 125px;
	  transform: translate(-25px, -25px)
	  
	}

	.child2 {
	  width: 50px;
	  height: 50px;
	  background-color: green;
	}
	
	.child3 {
	  width: 50px;
	  height: 50px;
	  background-color: yellow;
	}
```
- fixed positioning - the element is positioned according to the viewport and is removed from the document flow entirely; creates a new stacking context; the element will remain centered where it was on the window
- sticky positioning - sticky always stick to the closest scrollable ancestor; it creates a new stacking context and requires top, bottom, left or right to define at which offset it should begin sticking
	``` html
	<body>
	  <header>I'm a header</header>
	  <main>
	    <p>This is the main content of the website</p>
	  </main>
	</body>
```

	```scss
	body {
	  overflow: auto;
	}

	header {
	  padding: 10px;
	  background-color: blue;
	  color: white;
	}

	main {
	  height: 1200px;
	  background-color: red;
	}

	p {
		margin: 0;
	}
```

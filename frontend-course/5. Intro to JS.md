1. History - JavaScript is a scripting language, created to enhance the capabilities of a web page. It can run not only in the browser, but in a server environment, thanks to Node.js. It is a single-threaded language, without a strict type system.
2. Variable declaration
	- variables are declared via **let** and **const**. const is used for variables, which will not change their values, whereas let allows us to assign a new value after initialization. Both are **block-scoped**
	- const must be initialized the moment it is declared!
	- Block-scoped variables cannot be redeclared
	```js
	// Not possible!
	let myLet = 10;
	let myLet = 15;
	```
	- there also exists the old way of declaring variables, which is **var**. However var is currently not recommended due to its **global scoping** and something called **hoisting**. We can also redeclare var, unlike let and const, which can lead to further complications.
	```js
	// Will not error out
	console.log(5 + myVar);
	var myVar = 8;

	// Errors out
	console.log(5 + myLet);
	let myLet = 8;

	console.log(5 + myConst);
	const myConst = 8;
	```
3.  Data types
	- primitives - number, string, boolean, null, undefined. These are immutable and are copied by value; they are also stored in the stack.
	- undefined is assigned to uninitialized variables (absence of a value), whereas null simply marks a "blank" value
	```js
	const myNum = 5;
	const myDoubleDigitNum = 5.5;
	const myString = 'Hello World!';
	const myBool = true; // false

	let a = 5;
	let b = a; // b = 5, a = 5
	b = 10; // b = 10, a = 5
	```
	- reference types (composite types) - arrays and objects. These are mutable - their values can change and are copied by reference. They are stored in the heap.
	```js
	const myArr = [1, 2, true, 'what'];
	myArr.push(5); // [1, 2, true, 'what', 5]
	const myArr2 = myArr;
	myArr2.push(6); // myArr = [1, 2, true, 'what', 5, 6]

	const myObj = { name: 'Lenko', job: 'Frontend Developer' };
	const myObj2 = myObj;
	myObj2.job = 'Village Idiot'; // myObj = { name: 'Lenko', job: 'Village Idiot' }
	```
4. Basic operators
	- arithmetic - + , - , / , *
	- increment / decrement - ++ , --
	- assignment - =
	- logical - && , || , !, ??
	- comparison operators - == , === , !== , != , > , >= , < , <=
	```js
	const fiveNum = 5;
	const fiveString = '5';

	// Loose equality - won't account for types
	console.log(fiveNum == fiveString); // true
	// Strict equality - will check types as well
	console.log(fiveNum === fiveString) // false

	// Nullish coalescing operator
	console.log(null ?? 'Not a null'); // 'Not a null'
	```
	- typeof operator - a way to check the type of a variable and implement type guards
	```js
	const myNum = 42;
	// Typeof returns a string representation of the type
	console.log(typeof myNum); // 'number'

	// Type guards
	if(typeof myNum === 'number'){
		// do some calculations
	}

	if(typeof myNum !== 'undefined'){
		// do something
	}

	// Functions, arrays, objects
	function myFunc() {}
	console.log(typeof myFunc); // 'function'
	const myObj = {};
	console.log(myObj); // 'object'
	const myArr = [0,1];
	console.log(typeof myArr); // 'object'

	// NB! Check MDN for backstory
	console.log(typeof null); // 'object'
	const emptyVar = null;
	console.log(typeof emptyVar) // 'object'

	```

6. Functions
	- function declarations and function expressions
	```js
	// Function declaration
	function square(num) {
		return num * num;
	}

	// Function expression without or with name
	// Named function expression can refer to themselves
	const expressiveSquare = function(num) {
		return num * num;
	}
	const expressiveSquare2 = function sqr(num) {
		return num * num;
	}

	// All are invoked the same
	square(2)
	expressiveSquare(2);
	expressiveSquare2(2);
	```
	- reference types will be mutated
	```js
	const myObj = {
	    name: 'Lenko',
	    job: 'Frontender'
	};

	const myArr = [2, 5];

	const mutateMyObj = function (object) {
	    object.name = 'Shmenko'
	};

	const mutateMyArr = function (arr) {
	    arr[0] = 1;
	    arr[1] = 3;
	}

	mutateMyObj(myObj);
	mutateMyArr(myArr);

	console.log(myObj.name); // { name: 'Shmenko' }
	console.log(myArr); // [ 1, 3 ]
	```
	- arrow functions - a short way to write out anonymous functions
	``` js
	const arrowSquare = (num) => num * num;

	const arrowSquare = (num) => {
		return num * num;
	}
	```
	- the this keyword - the this keyword refers to different things at different points in our programs; usually, the global this when ran in the browser points to the window object (in the case of a function, it refers to the function object itself)
		Arrow functions do not have a this context!!!
	```js
		console.log('Global this: ', this);
		
		function myFunction() {
		    console.log('myFunction', this);
		}
		myFunction();

		const myFunc = function() {
		    console.log('myFunc', this);
		}
		myFunc();

		const myFunc2 = function funky() {
		    console.log('myfunc2', this);
		}
		myFunc2();

		const arrowFunction = () => {
		    console.log('arrowFunction', this);
		}
		arrowFunction()
	```
	- function hoisting - does not work with function expressions!
	```js
		// Function definition is hoisted to the top
		console.log(square(5));
		function square(num) {
			return num * num;
		}

		// Doesn't work, function expressions or arrow functions are not hoisted
		console.log(mySquare(5));
		const mySquare = (num) => num * num;
	```
	-  functions passed to other functions are called callbacks!
7. Arrays
	- as we saw, arrays can hold values of different types, however we do not recommend this, as it will lead to confusion
	- 0-indexed, overwriting elements as we have shown; do not have a fixed length!
	```js
	const myArr = [1];
	myArr[1] = 2;
	console.log(myArr); // [1,2]
	```
	- push and pop - add and remove elements
	``` js
	const myArr = [1];
	myArr.push(15); // pushes a new value --> [ 1, 15 ]
	myArr.pop(); // removes the last value --> [ 1 ]
	```
	- array methods
	``` js
	// map -> Returns a new array after modifying the items of the old one via a provided function
	const myArr = [1, 2, 3];
	console.log(myArr); // [ 1, 2, 3]
	const doubledArr = myArr.map((element) => element * 2);
	console.log(doubledArr); // [ 2, 4, 6 ]
	
	// slice -> Returns selected items from the old array into a new one based on the interval provided by the parameters
	const myArr = [ 1, 2, 3, 4, 5];
	const sliced1 = myArr.slice(2); // [ 3, 4, 5 ]
	console.log(sliced1);
	const sliced2 = myArr.slice(1, 3) // [ 2, 3 ]
	console.log(sliced2);
	
	// splice -> Modifies the original array according to the interval (and the new value) provided by the parameters
	const months = ["Jan", "March", "April", "June"];
	months.splice(0, 2); // [ "April", "June" ]
	console.log(months);
	months.splice(1, 1, 'August') // [ "April", "August" ]
	console.log(months);
	
	// reduce -> Run some function on all the elements, whilst using its previous calcualtion as part of it. If no initial values is provided, the first element will be considered the initial value.
	const myArr = [ 1, 2, 3];
	const reduceResult = myArr.reduce((accumulator, currentValue) => accumulator + currentValue); // 6
	console.log(reduceResult);
	
	const initValue = 10;
	const reduceResult2 = myArr.reduce((accumulator, currentValue) => accumulator + currentValue, initValue); // 16
	console.log(reduceResult2); 
	```
	- spread operator
	```js
	const myArr = [ 1, 2, 3 ];
	const copiedArr = [ ...myArr ];
	const refArr = myArr;
	console.log(myArr); // [ 1, 2, 3 ]
	console.log(copiedArr); // [ 1, 2, 3 ]
	console.log(refArr) // [ 1, 2, 3 ]
	console.log(myArr === copiedArr); // false
	console.log(refArr === myArr) // true
	```
8. Objects
	- properties
	```js
	const lenko = {
	    name: 'Lenko',
	    job: 'Frontend'
	}
	// Accessing properties
	console.log(lenko.name); // 'Lenko'
	// Assigning after initialization
	lenko.age = 23;
	console.log(lenko); // { name: 'Lenko', job: 'Frontend', age: 23}

	delete lenko.job;
	console.log(lenko) // { name: 'Lenko', age: 23}
	```
	- object methods - we can assign methods as properties; this has its own context in objects! - it refers to the object itself
	``` js
	const lenko = {
	    name: 'Lenko',
	    job: 'Frontend',
	    work: () => console.log('Working...'),
	    printFullName:function () { 
	    console.log(this.name + ' Ivanov') } // stay clear of arrow functions if you want to access this in the context of the object!
	}
	lenko.work(); // 'Working...'
	lenko.printFullName(); // 'Lenko Ivanov'
	```
	- spread operator can be used on all iterables, objects included
	```js
	const myObj = { name: 'Lenko', job: 'Frontend'};
	const refObj = myObj;
	const copyObj = { ...myObj };
	console.log(myObj);
	console.log(refObj);
	console.log(copyObj);
	console.log(myObj === refObj); // true
	console.log(copyObj === myObj); // false
	```
	-  keep in mind that the spread operator won't copy referential values nested inside!
	```js
	const myObj = {
	name: 'name',
	list: [1, 2, 3],
	};

	const obj = { ...myObj };
	obj.list.push(42);

	console.log(obj.list); // [1, 2, 3, 42]
	console.log(myObj.list); // [1, 2, 3, 42]
	```
	-  built-in object methods
	```js
const lenko = {
	  name: 'Lenko',
	  job: 'Frontend',
	  work: () => console.log('Working...'),
	  printFullName: function () { console.log(this.name + ' Ivanov')}
}

// Returns all keys 
console.log(Object.keys(lenko)); // [ 'name', 'job', 'work', 'printFullName' ]

// Returns all values
console.log(Object.values(lenko)); // [ 'Lenko', 'Frontend', [Function: work], [Function: printFullName] ]

// Returns key-value pairs
console.log(Object.entries(lenko)); // [ [ 'name', 'Lenko' ],[ 'job', 'Frontend' ], [ 'work', [Function: work] ], [ 'printFullName', [Function: printFullName] ] ]

// Copy properties into a new object
const newLenko = {};
Object.assign(newLenko, lenko);
console.log(newLenko);
console.log(newLenko === lenko); // false

lenko.age = 23;

// Cannot add or modify properties
Object.freeze(lenko);
lenko.hobbies = [ 'Books', 'Warhammer' ];
lenko.age = 24;
console.log(lenko);

// Cannot add properties but can modify existing ones
Object.seal(lenko);
lenko.hobbies = [ 'Books', 'Warhammer' ];
lenko.age = 24;
console.log(lenko);
```

1. HTML
	- opening/closing tags
	- attributes
	- most common tags - div, span, a, img, ul, ol, li, body, head
		- head - contains the metadata of the page - titles, stylesheets and JS scripts - title, style, link, script tags
		- body - contains all HTML content
		- form, label, button, select, input
		- h1-h6, p
2. Semantic HTML
	- the div soup issue
	- write meaningful html - better SEO performance, greater accessibility for screen readers 
	- header, footer, nav, aside, main, section, article
	- emphasis on elements - em and strong tags, b not for that
	- cannot nest li, p elements - tags automatically close each other
	- common element misuse - i tag as icon tag, whereas it semantically is used to create an italic element
	- significance of the button element - can house content inside in contrary to an input with type "button" for example
	- ul in nav - you can put ul/ol in the nav element, however this could obstruct some screen readers (a list with 10 links is simply a list of 10 items, the reader does not immediately say "this is a navigation with links") --> accessibility workshop - not in this one
	- article - a part of the webpage that is meaningful on its own (like a real life article); section - a block of related contents

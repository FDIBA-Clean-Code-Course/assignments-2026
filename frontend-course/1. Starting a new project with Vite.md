1. Initializing a project with Vite

- What is Vite? - Vite is a bundling tool that also provides an out-of-the-box dev server, which greatly enhances DX
- Bundling is an essential step in preparing frontend code for production - i.e. it is the process of linking all related files in chunks of code that will be served during the respective page load
- Bundling with Vite includes the process of tree-shaking - only the used dependencies are included in the final build + dead code is removed from the build
- Vite offers Hot Module Reloading out of the box - this lets the bundler reload only a certain module after a change is detected, preserving current app state and offering a great DX in terms of seeing the reflected changes
- Vite has out-of-the-box support for TypeScript, JSX/TSX extensions (relevant for React), Vue and other frameworks, as well as support for the most popular CSS preprocessors
- Scaffolding with Vite

```bash
npm create vite@latest --> manually configure a template
npm create vite@latest my-vue-app -- --template vanilla --> automatically apply a pre-made template
```

2. The vite.config file

- this provides us a way to configure the dev server, build options, etc.
- Vite is greatly optimized to reduce dev server startup, which works great in smaller project, because  Vite essentially loads the modules on the goal - dependencies are not pre-bundled. This can quickly become a nightmare in bigger projects
- Some ways to mitigate that are to either lazy load resources, or to specify imports - most imports are not specific enough, which results in the whole dependency module being imported
- what is chunking? - Chunking is the operation of code splitting - it is the process of only serving relevant code whenever it is needed --> demonstrate chunking
- a multipage application with Vite - serve the page inside of the public folder

```js
import { defineConfig } from "vite";

export default defineConfig({
server: {
 port: 3000,
 open: true,
},
build: {
 outDir: 'build',
 rollupOptions: {
  output: {
   manualChunks: {
   libs: ['date-fns']
  }
 },
 input: {
  main: 'index.html',
  dummy: "dummy.html"
 }
}
}
});
```

3. Prettier&ESLint setup
- Install the VSCode plugins - Eslint microsoft, Prettier prettier
- Install the packages

```bash
npm i eslint @eslint/js eslint-config-prettier eslint-plugin-prettier prettier --save-dev
```

- npm init @eslint/config
- integrate prettier with eslint

```js
import { format } from 'date-fns'

export default [
{ ignores: ['build/']},
{languageOptions: { globals: globals.browser }},
pluginJs.configs.recommended, // ESLint JS rules
{
plugins: {
 prettier: eslintPluginPrettier // Adds Prettier as an ESLint plugin
},
rules: {
'prettier/prettier': 'error', // prettier formatting shown as ESLint error
'no-unused-vars': 'warn', // warning for unused vars
 eqeqeq: 'error', // errors unless using ===
 curly: 'error', // errors unless all blocks have curly braces
}
},
configPrettier // disables conflicting ESLint rules

]; 
```

- npm scripts

```json
"lint": "eslint",
"format": "prettier --config .prettierrc --write ."
```
